# Step 04 Designer Review Packet

This checkpoint packages the work map for designer and front-end review before anyone reads raw `.tg`.

Read in this order:

1. `work-map.md` for the executive summary, screen inventory, layout/region map, widget work leaves, component map matrix, gaps, and agent handoff commands.
2. `design-coverage.md` for the widget-first component matrix and review work.
3. `screen-slice.md` for what an agent sees before changing the Intake Queue.
4. `widget-slice.md` for the data-grid mapping and review boundary.
5. `ui-realization-report.json` for deeper generated/maintained proof data.

The packet should make the UI work map concrete for a designer without requiring them to inspect parser records first.
